# Roadmap

Steplight is currently in its first public alpha. The goal is to become the local-first inspection tool for agent traces, tool-driven workflows, and LLM execution debugging.

## Current Focus

- Keep the core parsing surface reliable and easy to extend
- Improve the quality of diagnostics and summaries
- Make the local debugging loop feel fast and trustworthy
- Keep the product simple enough to use without setup overhead

## Near Term

### Parser Coverage

- Expand OpenAI trace support beyond the first supported response shapes
- Improve LangChain coverage for more callback and event variants
- Add more real-world MCP log patterns and edge cases
- Make generic mapping more flexible for nested and irregular event payloads

### Diagnostics

- Add richer timing diagnostics for stalled or imbalanced runs
- Add better retry analysis across tool and model steps
- Add context-window pressure warnings before the run becomes expensive or unstable
- Add stronger silent-failure detection when errors are swallowed by orchestration layers

### UX

- Improve the Textual inspector layout for larger traces
- Add better navigation and filtering for long timelines
- Improve HTML reports for sharing with teammates
- Add more polished examples, screenshots, and sample traces

## Next

### Analysis

- Run-to-run comparison for trace diffs
- Better cost and token attribution across steps
- Trace-level summaries for CI and release gates
- Optional rule configuration from a project file

### Extensibility

- Custom diagnostic rules without patching core code
- Cleaner adapter extension points
- Project-level configuration for thresholds and formatting

### Integrations

- Better interoperability with other tools in the same portfolio
- Tighter workflows around evaluation, migration, and observability
- Easier ways to inspect traces generated by production agents

## Longer Term

- Streaming or live trace inspection
- Plugin system for adapters and rules
- Trace diffing and trend analysis across many runs
- Deeper ecosystem support for common agent frameworks

## Guiding Principles

- Local-first by default
- No mandatory cloud dependency
- Terminal-native, with a strong non-interactive story
- Extensible through plain Python
- Useful on day one, not only after a complex setup
